# 安全检查记录

## 已做安全措施

- 用户密码使用 bcrypt 哈希，不保存明文密码
- JWT 设置过期时间
- 管理员接口使用权限校验
- 草稿文章只能管理员读取
- 点赞和收藏使用唯一约束，避免重复刷同一文章
- 评论内容限制最大长度
- 题库接口不返回正确答案，提交后才返回解析
- 安全小游戏必须完整答题后才计分，避免提交部分答案刷分
- 生产环境禁止使用默认 `SECRET_KEY` 和默认管理员密码
- React Markdown 不启用原始 HTML 渲染，降低 XSS 风险
- Nginx 添加基础安全响应头
- `.env`、数据库文件、缓存文件不进入 Git

## 仍需上线前处理

- 修改 `docker-compose.yml` 中所有默认密码
- 备案完成后配置 HTTPS
- 将 `CORS_ORIGINS` 改成真实域名
- 云服务器安全组只开放必要端口：临时访问保留 80，SSH 22 限制为本人 IP，删除 Linux 不需要的 RDP 3389
- 数据库密码使用强随机值
- 定期备份 PostgreSQL 数据卷
- 如要长期生产使用，建议把 JWT 存储从 `localStorage` 改成 HttpOnly Cookie
